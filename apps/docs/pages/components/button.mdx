# Button

Styled clickable element with multiple variants for different semantic contexts.

## Usage

```tsx
import { Button } from '@ds-foundation/react';

<Button>Click me</Button>
<Button variant="solid" colorScheme="danger">Delete</Button>
<Button variant="outline">Cancel</Button>
<Button size="sm">Small</Button>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"solid" \| "outline" \| "ghost" \| "link"` | `"solid"` | Visual style |
| `colorScheme` | `"primary" \| "success" \| "warning" \| "danger" \| "neutral"` | `"primary"` | Semantic color |
| `size` | `"xs" \| "sm" \| "md" \| "lg" \| "xl" \| "icon"` | `"md"` | Size preset |
| `isLoading` | `boolean` | `false` | Shows spinner; disables button |
| `loadingText` | `string` | — | Replaces label text while loading |
| `leftIcon` | `React.ReactElement` | — | Icon rendered before label |
| `rightIcon` | `React.ReactElement` | — | Icon rendered after label |
| `asChild` | `boolean` | `false` | Render as child element via Radix Slot |
| `...rest` | `ButtonHTMLAttributes` | — | Forwarded to the `<button>` element |

## Accessibility

- Uses native `<button>` for built-in keyboard navigation and ARIA support
- Add `aria-label` when button content is icon-only
- Use `aria-pressed` for toggle buttons that switch between active and inactive states
